PsiFS: Frequently Asked Questions

Unsuitable Tar File Extractor

Why are the file names in a backup corrupt?

Backup files produced by PsiFS use the fltar RISC OS variant of tar files. This stores filenames using RISC OS rather than UNIX conventions. Unfortunately, some tools expect the opposite convention, so end up with corrupt the filenames.

Recent versions of !SparkFS operate correctly, but earlier versions get the filenames wrong. If problems are experienced then contact David Pilling for an upgrade to the latest version.

Alternatively, Frank Lancaster's public domain command line tar utility, or the *PsiFSTar command may be used without problems.
