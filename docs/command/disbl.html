PsiFS: *Commands

*PsiFSDisable
*PsiFSDisable
Disable use of the configured block driver

Syntax:*PsiFSDisable [-abort]

Parameters:-abort disconnect immediately without flushing data

Use: *PsiFSDisable closes any active connection and ends use of the configured block driver. This allows other programs to use the serial port.

The -abort switch does not wait for data to be flushed, so may result in data being lost. It should only be used as a last resort.

No action is taken if the block driver is already disabled.


Example:*PsiFSDisable

Related commands:*PsiFSDriver, *PsiFSEnable, *PsiFSListDrivers, *PsiFSStatus
